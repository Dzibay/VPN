# Подорожник VPN — визуальные нормы

Фирменный стиль строится на логотипе: чёрный фон, круг, два оттенка зелёного (мята + бирюза), сочетание «природа + сигнал / связь».

## Цвета

| Роль | Тёмная тема (по умолчанию) | Светлая тема |
|------|----------------------------|--------------|
| Акцент | `#58D68D` (мята) | `#1D9A5C` (насыщенный зелёный для читаемости) |
| Второй акцент / градиент | `#45B39D` (бирюза) | `#1A7A4A` |
| Фон | `#000000` / градиент к `#050A08` | `#F2F7F4` |
| Поверхности | `#0C100E` | `#FFFFFF` |
| Текст | `#E8F4EC` заголовки, `#A8B8B0` основной | тёмно-зелёно-серый набор в `:root` |
| Ошибка | кораллово-красный | `danger` в токенах |

Семантика: **не** смешивать с фиолетовыми/синими оттенками в UI — для графиков используются разные цвета рядов, но «бренд-заливка» — зелёная шкала (мята / бирюза).

## Типографика

- **Заголовки:** Sora (500–700).
- **Текст интерфейса:** Manrope (400–700).
- **Код / моно:** системный stack (`ui-monospace`, Cascadia, Consolas).

Межбуквенно: лёгкое ужатие у H1; базовая высота строки ~1.55.

## Компоненты

- **Скругления:** `--radius` 10px, крупные CTA / карточки 12px.
- **Кнопка primary:** градиент `мята → бирюза` (в светлой теме — градиент от `--accent` к `--accent-muted`), тень от `--accent-glow` при необходимости.
- **Карточки и навигация:** полупрозрачные поверхности, тонкая граница `--accent-border` / `--card-border`.
- **Фокус:** кольцо `--focus-ring` (зелёный полупрозрачный).

## Логотип

Файл: `public/icons/podorozhnik-logo.png`. В шапке: круг 40×40, фон круга чёрный, тень `var(--accent-glow)`.

## Технически

Все токены задаются в `src/style.css` через CSS custom properties. Светлая тема подключается при `prefers-color-scheme: light`.

## SEO и метаданные

- `robots.txt`, `sitemap.xml`, `.well-known/security.txt` генерируются при `vite build` (и отдаются в `npm run dev` через middleware). Базовый URL: сначала `VITE_PUBLIC_SITE_URL` (полный URL), иначе `SITE_ADDRESS` (домен без схемы, как в `deploy/.env`) → к нему добавляется `https://`. В Docker для сервиса `nginx` в `docker-compose` передаётся тот же `SITE_ADDRESS`, что и для Caddy.
- Иконки и манифест: каталог `public/icons/` (`favicon*`, `apple-touch-icon.png`, `web-app-manifest-*.png`, `site.webmanifest`, `podorozhnik-logo.png`).
- В `index.html`: description, canonical, Open Graph, Twitter Card, JSON-LD `WebSite`.
- Почту в `security.txt` замените на рабочую перед продом.
